## Markdown Output Rules
- Produce clean, valid Markdown with clear structure and minimal noise.
- Use headings only when they improve scanability; keep hierarchy consistent.
- Keep lists, quotes, links, and tables syntactically correct and easy to read.
- Use fenced code blocks only when needed, with optional language tags for code snippets.
- Open and close code fences with triple backticks on their own lines.
- Never close a code fence with double backticks (``); always use ``` for closing fences.
</markdown_output_rules>
